## var 使用规范

**基本原则**：默认使用 `var`，现代 IDE 已提供完善的类型提示功能。

---

### ✅ 默认使用 var

几乎所有场景都应使用 `var`：

```csharp
// 构造函数
var player = new Player();
var list = new List<int>();

// 方法返回值
var damage = CalculateDamage(attacker, target);
var data = LoadConfig(path);

// 数值类型
var distance = Vector3.Distance(a, b);
var count = 0;
var isValid = true;

// foreach 循环
foreach (var player in players) { }

// LINQ 查询
var activePlayers = players.Where(p => p.IsActive).ToList();

// 元组解构
var (success, data) = TryParse(input);
```

---

### ❌ 需显式声明类型的场景

唯一需要显式声明的情况：**需要强调接口抽象时**

```csharp
// ✅ 使用接口类型声明
IList<Player> list = new List<Player>();
IEnumerable<Player> collection = GetPlayers();
IComponent component = new TransformComponent();

// ❌ 这会掩盖接口抽象
var list = new List<Player>();
var collection = GetPlayers();
```

## 代码换行规范

### 行长度限制

**单行最大长度**：120 字符，超出时必须换行。

### 换行位置

| 场景 | 规则 |
|------|------|
| 方法调用参数 | 逗号后换行 |
| 链式调用 | 点前换行，点在行首 |
| 运算符 | 运算符前换行（含 +, -, *, /, %, &, | 等） |
| 大括号 | 左大括号不换行（K&R 风格） |

```csharp
// 参数换行：逗号后换行
var result = SomeMethod(
    arg1,
    arg2,
    arg3
);

// 链式调用：点在行首
var result = someObject
    .Method1()
    .Method2()
    .Method3();

// 算术与逻辑运算符：运算符前换行
var result = longExpression1
    + longExpression2
    * longExpression3;

var isValid = condition1
    && condition2
    || condition3;

// 空合并与三元运算符：运算符前换行
var value = source
    ?? fallback1
    ?? fallback2;

var result = condition
    ? valueIfTrue
    : valueIfFalse;
```
