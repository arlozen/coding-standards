# 单元测试规范

## 基本原则

单元测试的目的是**验证业务逻辑**，提供快速反馈，支持重构。良好的测试应该：
- 测试**公共行为**而非实现细节
- **独立且可重复**运行
- 作为**活的文档**展示代码预期行为

---

## 组织结构

### 程序集格式

每个模块单独一个Tests程序集：

```
[ModuleName]/
    ├── Tests/
    │   ├── Runtime/           # 运行时测试
    │   │   └── Test[Category].[Feature].cs
    │   └── Editor/            # 编辑器测试
    │       └── Test[Category].[Feature].cs
```

### 命名规范

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 测试类 | `Test[Category][Feature]` | `TestSerializationBinary` |
| 测试方法 | `MethodName_Scenario_ExpectedResult()` | `GetPlayerById_ValidId_ReturnsPlayer` |
| 命名空间 | `{TestedNamespace}.Tests` | `EasyToolkit.Serialization.Tests` |

### Assembly Definition

**Runtime 测试**: `[ProjectName].[ModuleName].Tests.asmdef`
- `includePlatforms: []`（所有平台）
- `overrideReferences: true`
- `defineConstraints: ["UNITY_INCLUDE_TESTS"]`

**Editor 测试**: `[ProjectName].[ModuleName].Editor.Tests.asmdef`
- `includePlatforms: ["Editor"]`
- `overrideReferences: true`
- `defineConstraints: ["UNITY_INCLUDE_TESTS"]`

## 测试结构

### AAA 模式

使用 **Arrange-Act-Assert** 模式组织测试：

```csharp
[Test]
public void Add_TwoPositiveNumbers_ReturnsCorrectSum()
{
    // Arrange: 设置测试数据
    var calculator = new Calculator();
    const int a = 5;
    const int b = 3;

    // Act: 执行被测试的方法
    var result = calculator.Add(a, b);

    // Assert: 验证结果
    Assert.AreEqual(8, result);
}
```

### Region 分组

使用 `#region` 按功能分组：

```csharp
[TestFixture]
public class TestSerializationBinary
{
    #region Serialize/Deserialize

    [Test]
    public void SerializeDeserialize_Integer_ReturnsOriginalValue() { }

    #endregion

    #region Edge Cases

    [Test]
    public void Serialize_NullValue_ThrowsArgumentNullException() { }

    #endregion
}
```

## 测试方法命名

**格式**: `[MethodName]_[Scenario]_[ExpectedResult]()`

| 部分 | 说明 | 示例 |
|------|------|------|
| MethodName | 被测试的方法名 | `GetPlayerById` |
| Scenario | 测试场景或条件 | `WithValidId`, `WhenPlayerDoesNotExist` |
| ExpectedResult | 预期结果 | `ReturnsPlayer`, `ThrowsException` |

**示例**:

```csharp
// ✅ Good: 清晰描述测试场景
[Test]
public void GetPlayerById_ValidId_ReturnsPlayer() { }
[Test]
public void GetPlayerById_InvalidId_ReturnsNull() { }
[Test]
public void AddPlayer_DuplicateId_ThrowsInvalidOperationException() { }

// ❌ Bad: 名称不够具体
[Test]
public void Test1() { }
[Test]
public void Works() { }
```

## 需要测试的内容

### ✅ 应该测试

1. **公共 API** - 所有公共方法和属性
2. **边界条件** - null、空集合、最小/最大值
3. **异常路径** - 错误输入是否抛出正确的异常
4. **类型转换** - 接口转换、向下转型等
5. **状态变化** - 验证操作后对象状态的正确性
6. **复杂算法** - 多分支逻辑、业务规则
7. **事件回调** - 验证事件是否正确触发

### ❌ 不应该测试

1. **编译器保证的内容** - 类型继承、语法功能
   ```csharp
   // ❌ 无意义
   [Test]
   public void Player_IsAssignableFromIPlayer() { }
   ```

2. **简单属性** - 纯 Getter/Setter（无逻辑）
   ```csharp
   // ❌ 无需测试
   public int Id { get; set; }

   // ✅ 需要测试（有逻辑）
   public int Health
   {
       get => _health;
       set => _health = Mathf.Clamp(value, 0, 100);
   }
   ```

3. **私有实现细节** - 应通过公共 API 间接测试
4. **第三方库功能** - Unity 或已验证的库功能
5. **纯数据类** - 无行为的 DTO
6. **标记接口** - 仅用于类型标记的空接口

## 最佳实践

### 测试独立性

每个测试应该独立运行，不依赖其他测试的状态。使用 `[SetUp]` 初始化，`[TearDown]` 清理。

### 测试单一职责

每个测试只验证一个行为，便于定位问题：

```csharp
// ✅ Good
[Test]
public void AddPlayer_ValidPlayer_IncreasesCount() { }
[Test]
public void AddPlayer_ValidPlayer_TriggersEvent() { }

// ❌ Bad: 验证多个不相关行为
[Test]
public void AddPlayer_DoesEverything() { }
```

### 使用有意义的测试数据

```csharp
// ✅ Good: 测试数据表达意图
[Test]
public void CalculateDamage_With50Armor_ReducesDamageBy33Percent() { }

// ❌ Bad: 魔法数字
[Test]
public void CalculateDamage_SomeRandomNumbers_Works() { }
```

### 避免测试中的条件逻辑

```csharp
// ❌ Bad: 测试中有 if/else
if (condition) {
    Assert.AreEqual(expected1, result);
} else {
    Assert.AreEqual(expected2, result);
}

// ✅ Good: 拆分成独立测试
[Test]
public void Process_WhenConditionMet_ReturnsExpected1() { }
[Test]
public void Process_WhenConditionNotMet_ReturnsExpected2() { }
```

## 断言指南

| 断言方法 | 使用场景 |
|----------|----------|
| `AreEqual` | 验证相等 |
| `AreNotEqual` | 验证不等 |
| `AreSame` | 验证引用相同 |
| `IsNull` / `IsNotNull` | 验证 null |
| `IsTrue` / `IsFalse` | 验证布尔值 |
| `Throws` | 验证异常抛出 |

**浮点数比较** - 使用容差：

```csharp
Assert.AreEqual(expected, actual, 0.001f);
```

**异常测试**：

```csharp
var ex = Assert.Throws<ArgumentOutOfRangeException>(() =>
    obj.GetPlayerById(-1)
);
Assert.AreEqual("id", ex.ParamName);
```

## 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|------|------|----------|
| 测试实现细节 | 依赖私有字段或方法 | 测试公共 API |
| 脆弱的测试 | 与实现细节耦合太紧 | 测试行为而非步骤 |
| 忽略边界条件 | 只测试正常路径 | 覆盖 null、空值、极值 |
| 重复代码 | 多个测试有相同的 setup | 使用 `[SetUp]` 和 Builder 模式 |

## 覆盖率目标

| 代码类型 | 目标 | 说明 |
|----------|------|------|
| 核心业务逻辑 | 80%+ | 关键算法和业务规则 |
| 公共 API | 80%+ | 对外暴露的接口 |
| 工具/辅助方法 | 60%+ | 简单辅助功能 |

**注意**: 高质量测试 > 高覆盖率。测试有价值的行为，而非追求百分比。
